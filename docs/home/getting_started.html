Getting started

Where to get started if you want to use this tool

You can create your own static site with this tool. To get it up and running you will need to have Python 3.6 or higher installed with the following packages:

  • BeautifulSoup
  • Jinja2
  • Python-Markdown
  • pandas

It also uses several modules from the standard library (argparse, collections, configparser, io, pathlib, uuid).

Step 1: Download the repository

First download the repository from GitHub .

Step 2: Initial setup

The site builder needs some additional configuration files and folders to run that are not part of the repository. Rather, the scripts will create these files during their maiden voyage. Start out by running python build_structure.py which will:

  • Create content and output folders for your website.
  • Create a config.ini file.
  • Create a structure.xlsx file.

Then run python build_site.py which will:

  • Create a properties.ini file in the content folder.
  • Create a first .html page in the output folder.

Step 3: Give your website a name

Now that you have initialized the tool, go to the properties.ini file in the content folder. This file contains some basic properties about your site:

  • Name of the site.
  • Language (used in the html head).
  • Build version (should be an integer).
  • Text to be printed when a page is empty.
  • Footer information.

You can open and edit this file with your text editor. Give your site a name.

Step 4: Create a first structure for your site

The fourth step is to edit the site structure table which is stored as structure.xlsx in the content folder. Here you add the pages that will be part of the website. You also divide your site into sections, chapters and page groups. There are some rules for how to build up the structure table. Check the site structure table page for more details.

After you have created a site structure (or if you later on want to update your site structure) you run the build structure script : python script_build_structure.py which will add the newly added pages as .md files to the content folder.

Step 5: Fill your pages with content

The fifth step is to put some content in your pages. With a text editor you can edit the .md files in the content folder. You can use conventional markdown and add custom elements. See the cheatsheet for a quick overview of the syntax.

Step 4: Build your site

The fourth step is to build (a first version of) your website with the build site script by running python script_build_site.py This script will render your site as html into the designated output folder.

And that's it! For more details you can check the rest of the documentation.
