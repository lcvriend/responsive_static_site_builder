Build structure

This script builds or rebuilds the site structure within the content folder according to the site structure table in structure.xlsx

This script will do the following things:

Check the structure table

  1. Try to read the site structure from structure.xlsx in the content folder. If this fails it will create an empty structure table (with only a home page) and store it in the content folder.
  2. Check if any rows lack a page_id and if so create and add unique page_ids to these rows.

This script can add page_ids to rows in the table where they are missing but other than that the site structure table must be well formatted. At this point in time there is no other validation performed. This means specifically that the order for each page needs to be fully specified. Thus, make sure that section_oder, chapter_order and group_order are filled for each page. If there are no groups within a chapter or no chapters within a section the order value should be 1.

Delete .md files with unknown page ids

  1. Loop over the .md files in the content folder and for every file.
  2. Check the page_id
  3. If a page_id is found that is unknown to the site structure definition, then prompt the user to delete the file.

Files with page_ids that are not defined in the site structure are ignored when building the site. Thus they can be safely left within the content folder. However this may later lead to confusion when maintaining the site.

Update paths and filenames

Check if each path and filename follows this naming convention and if not rename them:

content folder / section_order + section / chapter_order + group_order + page_order + page_name

Another thing to note is that this script will organize the .md files into folders for convenience only. Where the .md files are stored has no bearing on how the site is actually built. You could move the files around and it would make no difference - as long as the site builder is able to find the relevant files.

Add any new page_ids as .md files

Check if all page_ids have an associated .md file. If this is not the case create the .md file following the naming convention above.

Save the updated structure table

Save the updated table in structure.xlsx
