Site structure table

The site structure table contains the structure of the site. It determines the hierarchy and order of the site pages.

A site that is built with this tool will have the following hierarchy:

Site

Sections

Chapters

Groups

Pages

This hierarchy is defined in the site structure table, which has the following columns:

Each row represents a page on the site and is indexed by its page_id

A valid site structure table must meet the following requirements:
- Each page is part of a section.
- Each page has a name.
- The order of every page is fully specified.

Sections

The sections can be used to arrange your site into main topics. The sections are shown in the navigation bar at the top of the page. Navigating to a section from the navigation bar will always take you to the first page of the section. Your site needs at minimum one section. Each page needs to be assigned to a section.

Chapters & groups

The chapters and groups are used to organize and structure the pages within a section into sensible units. Chapters and groups are shown in the navigation bar on the left side of the page. You do not need to use chapters or groups in your site. Leave the Chapter and Group fields empty if you do not want a page to be part of a chapter or group. You will need to fill the Chapter_order and Group_order fields.

Pages & cross-references

The pages are the units that contain actual content. You can assign a code to each page in the Code field. When the builder encounters a [code] in the text, it will convert it to an anchor tag that links to the page the code belongs to.

Order

Each order field within the structure table needs to have a value. The builder determines where the page fits within the total site order by taking into account all order fields. This is why every page should have a unique site order (which is the combination of section, chapter, group, page order).

All pages within a section should have the same Row order All pages within a chapter should have the same Chapter order All pages within a group should have the same Group order Within a group all pages should have a unique Page order
